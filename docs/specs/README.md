# Specs

`docs/specs/` is the home for numbered implementation specs.

Use this folder for:

- scoped build plans
- feature slices
- implementation sequencing
- spec documents that are expected to become code

Do not use this folder for:

- final implementation logs
- durable architectural decisions
- rough brainstorming notes
- long-running roadmap status

Related docs:

- `../ROADMAP.md` for future priorities
- `../decisions/` for rationale and durable choices
- `../builds/` for implementation history
- `../drafts/` for rough notes
