# Feature 5.3: Verification loop — agent result review

**Category:** Schema additions + verification logic in routing agent or standalone

## Done when

- When an execution agent completes work, the action status moves to `needs_review`
- Agent output is stored on the action (new `agent_output` column)
- Dashboard (Feature 1.2b) surfaces `needs_review` actions in a dedicated section
- You can approve (→ `done`), reject and reassign to human (→ `handed_back`), or retry (→ `agent_working`)
- If an execution agent fails, action moves to `needs_review` with the error captured
- This test works:
  ```
  Routing agent dispatches "Research Daikin mini-split pricing" to web_research agent
  → Agent completes, stores research summary in agent_output
  → Action status: needs_review
  → Dashboard shows it in "Agent work to review" section
  → You approve → status: done, completion_note auto-filled from agent_output
  ```

## Context

This is the feedback loop from the Claude Code diagram — "Verify Results." Without it, agents operate in the dark and you never know if the work was done or done well. The loop is:

```
[Execution Agent finishes] → needs_review → [You approve/reject] → done OR retry
```

This keeps you in control without requiring you to do the work.

## What to build

### Schema migration

```sql
-- Add agent output storage to actions
alter table actions add column agent_output text;      -- the work product from the agent
alter table actions add column agent_error text;        -- error message if agent failed
alter table actions add column agent_completed_at timestamptz;  -- when agent finished
```

### Status flow

```
pending_triage → pending_routing → agent_working → needs_review → done
                                                 → handed_back (human takes over)
                                 → needs_review (agent failed) → agent_working (retry)
```

### Verification in execution agents

Every execution agent, when it finishes work, must:
1. Store its output in `actions.agent_output`
2. Set `agent_status = 'needs_review'`
3. Set `agent_completed_at = now()`
4. If it failed: set `agent_error` with the error message

### Dashboard integration

Add to the Today view (Feature 1.2b):
- New section: "Agent work to review" — actions where `agent_status = 'needs_review'`
- Each card shows the action content + agent output preview
- Three buttons: Approve (→ done), Take over (→ handed_back, assigned_to: human), Retry (→ agent_working, re-dispatch)
- Approve auto-fills `completion_note` from `agent_output` summary

### Auto-approve option (future)

For low-stakes agent work (like routine notifications), allow a flag on the capability registry: `auto_approve boolean default false`. If true, skip `needs_review` and go straight to `done`. Don't build this yet — start with everything requiring review so you can calibrate trust.

## Dependencies

- 5.1 (triage agent) and 5.2 (routing agent) — this closes their loop
- 1.2b (today view) — for the dashboard integration
