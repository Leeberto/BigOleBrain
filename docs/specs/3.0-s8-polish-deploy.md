## Deployment

### Vercel setup

1. **Create Vercel project** linked to the Git repo, pointing at `apps/brain/` as the root directory
2. **Framework preset:** Next.js (auto-detected)
3. **Build command:** `npm run build` (or `next build`)
4. **Output directory:** `.next` (default)
5. **Node version:** 18.x or 20.x

### Environment variables

Set in Vercel project settings (not in code):

```
NEXT_PUBLIC_SUPABASE_URL=<project-url>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon-key>
```

No server-side secrets needed — the app uses the anon key + RLS for all data access.

### Domain

- Vercel will assign a `.vercel.app` URL automatically
- Optionally configure a custom domain if Lee has one (e.g., `brain.example.com`)
- Ensure HTTPS is enabled (Vercel does this by default)

### Supabase configuration

Verify that the Supabase project's auth settings allow the Vercel domain:
- **Site URL:** Set to the Vercel production URL
- **Redirect URLs:** Add the Vercel URL (for auth callbacks if using magic link)
- **CORS / Allowed origins:** Supabase typically handles this automatically with the anon key, but verify

## Error states

Create a shared `ErrorState` component if one doesn't exist:

```tsx
// components/shared/ErrorState.tsx
type ErrorStateProps = {
  message: string
  onRetry?: () => void
}
```

- Centered message with muted text
- "Try again" button if `onRetry` is provided
- Catches both network errors and Supabase query errors
- Friendly language: "Something went wrong" not "Error: PGRST116"

**Error boundaries:** Add a React error boundary at the layout level so the entire app doesn't crash if one component throws. The error boundary should show a simple "Something went wrong. Reload the page." message.

```tsx
// components/shared/ErrorBoundary.tsx
// Wrap around the main content in layout.tsx
```

## PWA manifest

Make the app installable to the home screen on iOS and Android.

### next.config.js or app/manifest.ts

```ts
// app/manifest.ts (Next.js App Router convention)
import type { MetadataRoute } from 'next'

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: 'BigOleBrain',
    short_name: 'Brain',
    description: 'Household hub + personal control plane',
    start_url: '/',
    display: 'standalone',
    background_color: '#ffffff',
    theme_color: '#ffffff',
    icons: [
      {
        src: '/icon-192.png',
        sizes: '192x192',
        type: 'image/png',
      },
      {
        src: '/icon-512.png',
        sizes: '512x512',
        type: 'image/png',
      },
    ],
  }
}
```

### App icons

Generate simple app icons (192px and 512px). Options:
- Use a simple SVG-based icon (brain emoji or a simple geometric mark) rendered to PNG
- Or use a placeholder solid-color square with "B" text — this can be replaced with a real icon later

Place in `public/icon-192.png` and `public/icon-512.png`.

### iOS-specific meta tags

Add to `app/layout.tsx`:

```tsx
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<meta name="apple-mobile-web-app-title" content="BigOleBrain" />
<link rel="apple-touch-icon" href="/icon-192.png" />
```

### No service worker needed for v1

A full offline-capable PWA with service worker is overkill. The manifest + meta tags are enough to make the app installable to the home screen, which gives it a standalone feel without the browser chrome. Offline support can be added later if needed.
