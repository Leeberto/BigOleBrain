# Feature 5.4: Execution Agent — web research (first agent)

**Category:** `supabase/functions/research-agent/index.ts`

## Done when

- `research-agent` Edge Function exists
- Accepts an action ID, reads the action content + linked thought context
- Performs web research using LLM with web search capability
- Stores a structured research summary in `actions.agent_output`
- Sets `agent_status = 'needs_review'`
- Registered in `agent_capabilities` table
- This test works:
  ```
  Action: "Research current pricing for Daikin mini-split systems"
  → Research agent gathers pricing info from web
  → Stores summary: models, price ranges, key considerations
  → agent_status: needs_review
  → You review and approve or modify
  ```

## Context

This is the first concrete execution agent — the one that proves the pipeline works end-to-end. Web research is the best starting point because:
- It's useful immediately (you have research-type actions right now)
- It requires only LLM + web search (no external integrations like Gmail)
- The output is text, easy to review
- Low risk if the agent gets it wrong (you just reject and do it yourself)

Once this works, the same pattern applies to every future execution agent: email drafting, deck building, scheduling, etc.

## What to build

### Edge Function

`supabase/functions/research-agent/index.ts`

Input: `{ action_id: string }`

Logic:
1. Fetch the action from the `actions` table
2. Fetch the linked thought (if `thought_id` exists) for additional context
3. Send to LLM with web search enabled:
   ```
   Research the following topic and provide a structured summary.
   
   Task: {action.content}
   Additional context: {linked thought content}
   
   Provide:
   - Key findings (3-5 bullet points)
   - Sources consulted
   - Recommended next steps
   - Any important caveats or uncertainties
   ```
4. Store LLM response in `actions.agent_output`
5. Set `agent_status = 'needs_review'`
6. Set `agent_completed_at = now()`

### Error handling

If the LLM call fails or returns garbage:
1. Set `agent_status = 'needs_review'`
2. Set `agent_error = error message`
3. Leave `agent_output` empty — the review step will show the error

### Capability registration

```sql
insert into agent_capabilities (name, description, handler_type, handler_config, required_integrations)
values ('web_research', 'Search the web and summarize findings on a topic',
  'edge_function', '{"function": "research-agent", "endpoint": "/functions/v1/research-agent"}', '{}');
```

### Follow the existing Edge Function pattern

Same as `open-brain-mcp`: Hono app, `x-brain-key` or `?key=` auth, environment variables for secrets.

## Dependencies

- 5.2 (routing agent) — dispatches work to this agent
- 5.3 (verification loop) — reviews this agent's output
- LLM API with web search capability
