## What Was Built

Verification loop for agent work: when an execution agent completes, actions move to `needs_review` where the user can approve, take over, or retry.

| File | Purpose |

| `apps/brain/src/lib/queries/agent-events.ts` | Replaced stub with real implementation querying actions with agent activity |
| `apps/brain/src/components/briefing/ReviewSection.tsx` | New homepage section showing actions needing review with approve/take-over/retry buttons |

| `apps/brain/src/components/actions/ActionDetail.tsx` | Added agent output/error display and approve/take-over/retry buttons in detail view |

## Status Flow

```
pending_routing → agent_working → needs_review → done (approve)
                                               → handed_back (take over)
                                               → agent_working (retry)
                                → needs_review (agent failed) → agent_working (retry)
```

## Key Decisions

- **ReviewSection on homepage** — placed between AgentSection and MealsSection, shows up to 5 cards with inline action buttons
- **Approve auto-fills completion_note** — first 500 chars of agent_output used as the completion note
- **Retry clears agent_error/output** — fresh start for the agent
- **Agent events now real** — replaced the stub `getRecentAgentActivity` with a query against actions where `agent_status` is not null
