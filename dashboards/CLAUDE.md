# dashboards Guidance

This folder is the standalone frontend template and community contribution surface.

Use `dashboards/` when:

- the frontend should remain reusable or deployable on its own
- the work is a template, example, or contribution
- the feature should not yet be absorbed into the canonical product app

If a dashboard feature is promoted into `apps/brain`, treat that as a product promotion and document it in the spec or build log.

Each runnable dashboard should keep:

- a README with purpose, environment, run, lint, and verify guidance
- an env example file
- a clear standalone deployment story
